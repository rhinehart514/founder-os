# Blueprint Gotchas

Built from the REB session (2026-03-24) + Anthropic skill best practices + JTBD practitioner research. Update when /blueprint fails in a new way.

## Job identification

- **Founder language, not customer language.** "AI front office" is founder. "Answer my phone when I'm on a job" is customer. Test: would [customer] say this exact sentence?
- **Too many jobs.** 12 jobs = haven't merged. Most products: 4-6 real jobs. Merge by switching trigger.
- **Jobs without dollars.** "Look professional" can't be priced. "$45K/year lost to missed calls" can.
- **Tasks as jobs.** "Update my website" = task. "Look credible when Googled" = job. Jobs = progress, not actions.
- **Missing job types.** Only listing functional jobs. Emotional ("stop worrying about missed leads") and social ("look like a real company") jobs drive switching too.
- **Value map before customer profile.** (Strategyzer trap) Always map customer jobs/pains/gains BEFORE proposing solutions.

## Value props

- **Feature-first.** "GBP-to-site in 15 minutes" = feature. "Website that stays current without you touching it" = value prop.
- **Buzzword contamination.** "AI-powered", "intelligent", "smart" = founder words. Strip all.
- **Can't be repeated.** >1 sentence = not a value prop. 10-second peer test.
- **Generic.** "Save time and money" = applies to everything = differentiates nothing.

## Framing

- **Inside-out.** Describing what it IS vs what it DOES.
- **Category creation too early.** New categories need millions in marketing. Anchor to known concepts.
- **One frame for all contexts.** Acquisition ≠ retention framing.

## Feature ideation

- **Features without jobs.** Can't name which job → kill it.
- **One solution per opportunity.** Torres rule: always generate 3 candidate solutions. Prevents premature convergence.
- **Empty kill list.** If everything survives, you're not thinking critically.
- **Fighting market leaders.** Don't build scheduling to compete with Calendly.
- **No riskiest-assumption check.** Every candidate solution has one assumption that, if wrong, kills it. Name it.

## Tier sequencing

- **MVP too big.** 8 features ≠ MVP. 3-5 features, ONE job.
- **Tier 1 doesn't match headline.** Framing says "we answer your phone" but Tier 1 is a website builder = disconnect.
- **No displacement math.** Every tier: "replaces [tools] costing $X/mo" → price: $Y/mo.

## Research

- **Vendor data as gospel.** Revenue loss numbers come from companies selling the fix. Note incentive.
- **One anecdote as evidence.** 1 Reddit post ≠ data. 50 = signal. Note sample.
- **Research without synthesis.** Raw findings ≠ blueprint. "Data says X, so build Y" = the job.

## Opportunity scoring

- **Treating qualitative estimates as survey data.** Pre-product: flag explicitly as "qualitative estimate from N interviews, not survey."
- **Skipping satisfaction scores.** Importance alone is misleading. A high-importance, high-satisfaction outcome = table stakes. You need both axes.

## Skill-level

- **SKILL.md too long.** Keep <100 lines. Push depth to references/.
- **Generating format from scratch.** Use patterns/ examples. Consistency > creativity for structure.
- **Not persisting output.** Always write blueprint-[project]-[date].md. Next session reads it instead of re-doing.
